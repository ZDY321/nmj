## 准备工作：确认本地代码状态

**💻 本地电脑**

1. 用终端打开你的项目目录（就是包含 `package.json` 的文件夹）。
2. 确认 `wrangler.toml` 文件还在根目录，打开检查内容。  
   如果没有或者丢失了，需要新建一个，填入以下内容：

```toml
name = "teacher-salary-tracker"
main = "src/index.ts"
compatibility_date = "2026-05-11"

[assets]
directory = "./dist"
binding = "ASSETS"
not_found_handling = "single-page-application"

[[d1_databases]]
binding = "DB"
database_name = "teacher_salary_tracker"
database_id = "replace-with-cloudflare-d1-database-id"
```

> 注意：`database_id` 目前先用占位符，等创建 D1 后再替换。

3. 确认所有代码已经推送到 GitHub 仓库（如果还没有，先 `git add .`、`git commit -m "准备部署"`、`git push`）。

---

## 第一步：在 Cloudflare 创建 D1 数据库

**🌐 Cloudflare 网页控制台**

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)。
2. 在左侧菜单点击 **Storage & Databases** → **D1 SQL Database**。
3. 点击右上角 **Create database**。
4. 弹出窗口中填入：
   - **Database name**：`teacher_salary_tracker`
   - **Location**：保持默认（Automatic 即可）
5. 点击 **Create**。
6. 创建完成后，进入该数据库的详情页，你会看到 **Database ID**，一串类似 `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx` 的 UUID。  
   **复制这个 ID**（后面要用）。

---

## 第二步：更新 wrangler.toml 并推送

**💻 本地电脑**

1. 打开项目根目录的 `wrangler.toml`。
2. 找到这一行：
   ```toml
   database_id = "replace-with-cloudflare-d1-database-id"
   ```
3. 把引号里的内容换成你刚才复制的 **Database ID**。  
   例如：
   ```toml
   database_id = "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
   ```
4. 保存文件。
5. 将修改提交到 Git：
   ```bash
   git add wrangler.toml
   git commit -m "更新 D1 database_id"
   git push
   ```

---

## 第三步：在 Cloudflare 创建 Worker 项目（连接 GitHub）

**🌐 Cloudflare 网页控制台**

1. 左侧菜单点击 **Workers & Pages**（不是 Workers 下面的内容，就是顶部那个入口）。
2. 点击右上角 **Create**。
3. 在创建类型选择中，**一定要选择 “Import a repository”**（不要选 Pages 或 Start from scratch）。
4. 系统会提示你授权 GitHub。授权后选择你的 `teacher-salary-tracker` 仓库。
5. 如果页面让你选择类型（Workers 或 Pages），务必选择 **Workers**。  
   *如果你看到的界面是“框架预设”、“构建输出目录”，说明错了，那是 Pages，请返回重新选择 Workers。*
6. 进入配置页面，填写这些字段：

| 字段 | 填写内容 | 说明 |
|------|----------|------|
| **项目名称** | `teacher-salary-tracker` | 会自动填入，不用改 |
| **构建命令** | `npm run build` | 不要有多余空格 |
| **部署命令** | `npx wrangler deploy` | 同样不要有多余空格 |
| **根目录** | `/` | 表示项目根目录 |

其他字段如“环境变量”、“变量名称”等一律留空，不要碰。  
API 令牌区域选择 **“创建新令牌”**，随便起个名字如 `deploy-token`，然后继续。

7. 点击底部的 **“保存并部署”**（或 “Create project”）。

> 此时 Cloudflare 会开始根据你的 GitHub 仓库构建并尝试部署，但**可能会失败**（因为 D1 数据表还没初始化，且 Secret 没设）。不要慌，失败是正常的，我们继续下一步。

---

## 第四步：设置 ADMIN_API_TOKEN Secret

**🌐 Cloudflare 网页控制台**

1. 在 Workers & Pages 中找到你刚创建的 `teacher-salary-tracker` Worker 项目，点击进入。
2. 点击顶部标签栏的 **Settings**。
3. 在左侧菜单中选择 **Variables and Secrets**（有些版本叫 Variables）。
4. 点击 **Add** 按钮，添加：
   - **Name**：`ADMIN_API_TOKEN`
   - **Type**：选择 **Secret**（一定要选 Secret，不要选 Plain text）
   - **Value**：粘贴一串你自己生成的长随机字符串（例如：`R1dTg9qH4YpX2mL8sA6vN3cZ7uP0bK5wJ8fE2hQ4rT6yU9iO1pL3mV5xS7aD0gF`）
5. 点击 **Save** 保存。
6. 保存后，你可能需要重新部署一次 Worker 才能让 Secret 生效。我们可以先做完初始化，再一起触发部署。

---

## 第五步：初始化 D1 数据表（本地执行唯一一次）

**💻 本地电脑**

1. 确保你的终端打开了项目目录。
2. 登录 Cloudflare 账号（如果之前没登录过）：
   ```bash
   npx wrangler login
   ```
   会自动打开浏览器进行授权。
3. 执行数据库初始化命令（假设数据库名称是 `teacher_salary_tracker`，迁移文件路径是 `./migrations/0001_initial.sql`）：
   ```bash
   npx wrangler d1 execute teacher_salary_tracker --remote --file=./migrations/0001_initial.sql
   ```
4. 看到类似 `Executed 5 queries in 6ms` 就表示成功。  
   **这一生只需要做一次，以后更新代码再也不用执行这个命令了。**

---

## 第六步：触发最终部署

**🌐 Cloudflare 网页控制台**（或自动部署）

因为我们已经修改了 `wrangler.toml` 并推送了代码，同时也添加了 Secret，Cloudflare 的 Git 集成应该会自动检测并重新部署。如果没有自动部署，可以手动触发：

1. 进入 Worker 项目的 **Deployments** 标签页。
2. 点击 **Deploy latest commit**（或类似按钮）。
3. 等待部署完成，看到日志中不再报 `database_id` 错误，并且有 “Success” 信息。

---

## 第七步：验证部署是否成功

**💻 本地电脑**（直接用浏览器访问）

1. 部署成功后，Cloudflare 会给你一个 Worker URL，类似于：
   ```
   https://teacher-salary-tracker.你的子域.workers.dev
   ```
2. 用浏览器打开这个地址，应该能看到页面正常显示（前端界面）。
3. 检查 API 是否正常：
   ```
   https://你的地址/api/health
   ```
   应该返回：
   ```json
   {"ok":true,"service":"teacher-salary-tracker"}
   ```
4. 检查管理员接口是否受保护（可选）：
   ```
   https://你的地址/api/admin/users
   ```
   应该返回 `401 Unauthorized`，说明 Secret 生效了。
